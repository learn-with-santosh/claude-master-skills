# Viral Tweet Generator

This skill helps you write tweets that get engagement—likes, retweets, replies, and follows. It uses proven frameworks, psychological triggers, and pattern recognition from viral tweets.

## When to use this skill

Use this skill when you want to:
- Turn an idea, topic, or draft into a viral tweet
- Generate tweet ideas from scratch on a given topic
- Improve an existing tweet for more engagement
- Create tweet threads that hold attention
- Understand *why* certain tweets go viral

## Input formats

The skill accepts:
- **Raw idea**: "I want to tweet about productivity tips for developers"
- **Draft tweet**: "Here's my tweet, can you make it better: [tweet]"
- **Topic only**: "Write a viral tweet about AI agents"
- **Existing tweet to analyze**: "Why did this tweet work? [tweet]"

## Output format

Every output includes two parts:

### 1. The Tweet

The optimized tweet ready to post. If multiple variations are useful for A/B testing, you'll see 2-3 versions.

### 2. The Analysis

A breakdown of why this tweet works:
- **Hook type**: How it grabs attention in the first line
- **Structure**: The framework used (list, story, contrarian, etc.)
- **Psychological triggers**: What makes people engage (curiosity, social proof, controversy, etc.)
- **Optimization notes**: Character count, emoji usage, hashtag strategy

## Viral Tweet Frameworks

### 1. The Contrarian Take
Challenge conventional wisdom. Pattern interrupt.
```
Everyone thinks X. They're wrong. Here's why...
```

### 2. The Listicle/Framework
Numbered lists, step-by-step guides, actionable tips.
```
5 mistakes [target audience] makes when [activity]:
1. ...
```

### 3. The Personal Story
Vulnerability, lessons learned, transformation.
```
I spent [time/money] learning [skill]. Here's what I wish I knew:
```

### 4. The Question/Engagement
Polls, open questions, debate starters.
```
Unpopular opinion: [controversial but defensible take]. Thoughts?
```

### 5. The Data/Social Proof
Stats, screenshots, receipts with commentary.
```
Analyzed [N] [things]. The results surprised me:
```

### 6. The Motivational Shift
Mindset change, aspirational, call to action.
```
Stop doing X. Start doing Y. Here's the difference:
```

## Writing Process

When generating or optimizing a tweet:

1. **Identify the core message** - What's the one thing you want to say?
2. **Choose the best framework** - Which structure amplifies this message?
3. **Write the hook** - First 50 characters determine if people read the rest
4. **Build the body** - Deliver on the hook's promise
5. **Add the landing** - Final line that drives action (follow, reply, think)
6. **Analyze** - Explain why this works using the framework above

## Character Limit

Keep tweets under 280 characters. If the idea needs more space, suggest a thread format.

## Hashtags

Use hashtags sparingly and strategically:
- 0-1 hashtags for most tweets (feels more native)
- Only include if they add discoverability in your niche
- Avoid hashtag stuffing

## Emoji Usage

- Use emojis as visual breaks or emphasis, not decoration
- 1-3 emojis max for most tweets
- Place at line starts for bullet-like effect
- Match emoji tone to content tone

## Niche Adaptation

This skill works across niches. Key considerations:
- **Tech/Startup**: Focus on insights, data, contrarian takes on trends
- **Marketing/Creator**: Emphasize actionable tips, case studies, personal wins
- **Finance/Business**: Lead with numbers, outcomes, specific results

## Thread Format

For longer ideas, use thread format:
```
[Hook tweet that stands alone]

[Supporting point 1]
[Supporting point 2]
[Supporting point 3]

[Summary/CTA]
```

Each tweet should be skimmable and add value independently.

## Safety Check

Before finalizing, consider:
- Could this be misinterpreted as hostile or offensive?
- Does it punch down or mock a protected group?
- Is the controversy productive or just inflammatory?

Adjust tone if needed while preserving the message.

## Examples

### Example 1: Transforming a raw idea

**Input**: "I want to tweet about how most people waste time in meetings"

**Output**:
```
Your meetings could be emails.

But here's the real problem: most emails could be no messages at all.

Before you schedule that "quick sync":
- What decision are we making?
- What info do we need?
- Can this happen async?

If you can't answer, don't book it.
```

**Analysis**:
- **Hook type**: Contrarian statement + escalation
- **Structure**: Problem → Reframe → Actionable checklist
- **Psychological triggers**: Validation (frustration with meetings), authority (clear framework), urgency (immediate applicability)
- **Optimization**: 247 characters, no emojis (clean/professional), no hashtags

### Example 2: Improving an existing tweet

**Input**: "Here's my tweet: I learned a lot about building startups. You need to talk to customers and iterate fast and not give up."

**Output**:
```
Version 1 (Direct):
I spent 2 years building a startup that failed.

Here's what I learned the hard way:
- Talk to customers before writing code
- Iterate fast, but not on the wrong thing
- Persistence > motivation

The cost of ignoring these: $500K and 800 days.
```

**Analysis**:
- **Hook type**: Personal story + specific stakes
- **Structure**: Setup → List → Punchy closer with numbers
- **Psychological triggers**: Social proof (specific numbers), curiosity (what else did they learn), authority (paid the tuition)
- **Optimization**: 268 characters, no hashtags (feels more authentic)
